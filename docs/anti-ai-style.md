## 一、用字黑名單（命中就換掉）

### A. 灌水開頭／教科書過渡詞
真人很少在每段開頭端出連接詞，AI 卻很依賴。**禁用**：
- 深入探討、深入挖掘、讓我們深入了解、值得注意的是、重要的是、必須記住
- 首先 / 其次 / 最後 / 然而 / 總而言之（段落結構過於對稱工整也是 AI 味）
- **宣告式「誠實／坦白」開場**：先誠實講一件事 / 老實說 / 坦白說 / 說真的 / 說實話 / 不得不說。真要誠實的人直接講那件事，不會先昭告「我要誠實了」，這是裝真誠的 AI 腔。
- **宣告式「我會說清楚／這篇會告訴你」導引**：我會說清楚 / 我會說明 / 這篇會告訴你 / 本文將介紹 / 接下來會講 / 帶你了解。本來就會把事情講清楚，不用先昭告「我要講了」，直接講內容即可。（註：「我會親自確認那幾則」這種描述**實際動作**的不算，要擋的是宣告「我要解釋」。）
- **宣告式「把話講在前面」開場**：先把話講在前面 / 話先說在前頭 / 醜話說在前面。要講的重點直接講，不用先預告「我要先講一件事」。
- **宣告式「舉例」開場**：舉個例子 / 舉個實際的例子 / 舉個我的案例 / 舉例來說 / 比如說（放句首當報幕）。要舉例就直接把例子講出來，不用先昭告「我要舉例了」。（註：例子融在句中的「像那條 LINE 流程就…」「我自己一條流程就是這樣：…」不算，要擋的是句首端出「舉例來說：」這種報幕。）
- **宣告式「先交代／先說明」開場**：先交代這個例子是什麼 / 先說明一下 / 先解釋一下背景。要交代就直接交代，不用先昭告「我要交代了」——刪掉這句、後面內容原封不動就是正解。（2026-07-04 實例：「先交代這個例子是什麼。IG 監控工具是…」→ 直接寫「這個 IG 監控工具是…」）
- **換段過場宣告（高頻犯規，使用者明確點名「講過很多次」）**：概念講完，來接真的 / X 都講完了，這段來聊 Y / 設定講完，來看實戰 / 接通之後，真正好玩的才開始 / 好戲在後頭。這種「先總結上一段、再預告下一段」的接縫句沒有資訊量，**段落之間不需要報幕**，直接寫下一段的實際內容即可。
- **改法**：直接講下一件事，不要端連接詞、不要宣告開場、不要在換段時總結＋預告；段落長短故意做不對稱。

### B. 浮誇強調詞（把小事講得驚天動地）
**禁用**：至關重要、關鍵的、樞紐的、不可磨滅（「留下了不可磨滅的印記」）、顯著的、凸顯了 / 強調了（拿抽象事實當主詞）。
- **給抽象事物套虛無的感官／情緒形容詞**：發現「有點刺眼」、數字「觸目驚心」、結果「血淋淋」、「發人深省」「耐人尋味」。發現不會刺眼、數字不會觸目，這是不符現實的硬套，最常見。
- **改法**：用具體數字或後果代替形容詞，「省 40 分鐘」勝過「至關重要」、「同事不用加班到八點」勝過「顯著提升」；直接講事實，「將近九成客戶一年只買一次」本身就夠力，不用再加「刺眼」。

### C. 社群情緒假詞（裝熟、空洞）
**禁用**：穩、撐、懂（懂的都懂 / 穩了 / 撐住）、接住 / 不繞 / 很現實（「這次換我來接住你的情緒」）、其實 / 很清楚 / 很簡單（當語氣助詞濫用）。
- **改法**：講具體場景與動作，不裝感性。

### D. 經典 AI 句型（抓包率最高）
**禁用**：
- 「這不是……而是……」（工整反轉金句）
- 「不僅是……更是……」（無限昇華意義）
- 「從……到……」假範圍（跨度過大、邏輯斷裂，如「從宇宙起源到現代行銷」）
- 破折號（——）濫用，尤其句尾補充——**被網友公認 99% 秒判 AI 的符號**
- 三段式排比（連列三個形容詞 / 「願妳勇敢、願妳溫柔、願妳永遠是自己」）
- 僵化結尾（In conclusion / Overall / 綜上所述）
- **殘句標語**：H2 或重點句寫成「問句？＋聳動斷句」，句子不完整又沒吃關鍵字（如「群發 EDM 為什麼都沒人打開？因為你對 2 萬人發同一封」——「發同一封」什麼沒講完）。真人寫標題會把句子收完整。
- **憑空宣告「是真的／不是假的」**：前文沒人質疑、沒設下「這是不是真的？」的問句，卻突然斷言「這兩個風險都是真的」「這擔心不是多餘的」「這問題確實存在」。AI 愛先替讀者立一個假想質疑、再出面證實，真人不會對沒被問的事自證為真。
- **「閃得掉／躲得掉／逃不掉」這類裝順口語**：用一個看似生動的動詞包裝「可以避免」（如「這兩個風險都閃得掉」「這問題躲不過」），讀起來像口語其實沒人這樣講，是 AI 硬找俏皮動詞的痕跡。**改法**：多半整句直接刪掉、進入下一句講具體做法即可；真要講就用平實的「都能避開／可以避免」。
- **「出手」當「操作」用的武俠腔**：「人唯一出手的一次」「只在最前面出手」這種把日常操作包成武俠動作的寫法，同屬硬找俏皮動詞（2026-07-04 被點名太 AI）。**改法**：換平實的「動手／要人操作」——「全程只有這步要動手」。
- **改法**：用「而」＋具體脈絡（「她哀悼的不是過去，**而是**回不去的童年」）；一句話只講一件事；H2／重點句寫成**完整句子**並把關鍵字鑲進去（「…因為你把同一封信發給了所有客戶」）；要講的事直接講，過渡句寧可刪掉也不要硬塞俏皮動詞。

### E. 空洞、模糊、格式殘留
**禁用**：空洞讚美（燈塔 / 見證 / 文化遺產中心）、模糊歸因（「批評者認為……」不指名）、Markdown 殘留（`##`、`**` 跑進正文）、異常工整的條列與大寫標題。
- **改法**：歸因要指名；讚美換成可驗證的事實。

### F. 抽象化、缺乏錨點
**禁用**：
- 概念名詞化：自我的探索、認同感的建構、內在小孩的療癒、效率的提升
- 缺乏錨點：某個午後、在忙碌的城市裡、認識一個朋友她、大幅提升、許多企業
- **改法**：名詞化換動詞口語（「自我的探索」→「找自己」；「效率的提升」→「少做一堆白工」）；抽象錨點換真實時間／地點／人名／數字。

## 二、EEAT 具體化（正面準則：真實 > 宏觀）

AI 味的根源是「統計上最安全＝空泛宏觀」。人味＝**活過的證據**。每篇都要做到：

1. **具體錨點取代概念**：真實的時間、地點、數字、客戶情境。
   - ❌「自動化能大幅提升行銷團隊效率」
   - ⭕「上週幫一個賣保健食品的客戶接 n8n，他原本每天早上花 40 分鐘把 GA4 數字貼進 Excel，現在 8 點打開 LINE 報表已經躺在那」
2. **動詞推動，不要堆名詞**：好句子靠動詞，名詞一多就軟綿綿（AI 受學術摘要訓練資料影響愛堆名詞）。
3. **節奏不對稱**：真人句子有長有短，不會每段一樣長、每組都排比。
4. **看得到「誰」，不是「人們」**：寫得出具體的人與動作，而不是抽象群體。
5. **真實，不是編造**：具體錨點必須來自真實素材（拉 material_log）。不能為了具體而編戲劇化假故事；匿名化的客戶情境要標示「示意」。（呼應 feedback_no_fake_anecdotes）

### 三個交付前檢驗法
- **能不能拍成電影？** 有溫度、氣味、畫面、動作才是真寫作；拍不出來＝AI 味。
- **唸出聲音會不會像作文範本？** 太順、太對稱、太工整 → 重寫。
- **看得到「誰」嗎？** 寫得出阿芳穿藍外套喝冰美式，而不是「一位朋友」。

## 三、各 skill 的差異套用

- **dev-diary / qa-article（長文）**：全部適用。重點在 A/B/D/E/F 與 EEAT 具體錨點；破折號、三段排比、「這不是…而是…」尤其要清。
- **threads-diary（社群口語）**：C（情緒假詞：穩／撐／懂／接住）與 D（這不是…而是…、破折號）是重災區，最優先擋；本來就走口語自嘲，更要避免裝感性。
- **linkedin-post（專業）**：B（浮誇強調詞）與 E（空洞讚美 / 模糊歸因）最該擋；專業感靠具體案例與數字，不靠形容詞。
