【這個範本在做什麼】
從 Google Sheets 讀取「最新一篇定稿」，自動同時發佈到
Facebook 粉專、Instagram、Threads，完成後 LINE 通知你，
並回寫 Sheet 標記「已發布」。一次寫稿，三平台同步上架。

------------------------------------------------------------
【一、Google Sheet 表單欄位】
------------------------------------------------------------
讀稿那張表至少要有這些欄(欄名要一字不差，工作流靠欄名抓)：
  - 文章標題
  - 文章內容
  - 圖片位址              (公開可讀的圖片網址)
  - 雲端圖片位址(轉化後)   (IG / Threads 優先用這欄)
  - FB圖片位址            (FB 優先用這欄，沒填就用上面)
  - 是否發布              (工作流發完會自動回填「是」)

注意：圖片一定要是「公開網址」，FB / IG / Threads 才抓得到，
私人雲端連結會失敗。

------------------------------------------------------------
【二、要替換的佔位符(全部換成你自己的)】
------------------------------------------------------------
「讀取上架文章表單」與「回寫是否發布」兩個 Google Sheets 節點：
  YOUR_GOOGLE_SHEET_ID    → 你的試算表 ID
  YOUR_SHEET_NAME_OR_GID  → 你的分頁名稱或 gid
  並接上你自己的 Google Sheets 憑證

「發文token」節點(Set 節點，集中放金鑰)：
  YOUR_META_ACCESS_TOKEN     → Meta(FB/IG)的 access token
  YOUR_THREADS_ACCESS_TOKEN  → Threads 的長期 access token
  YOUR_FB_PAGE_ID            → 你的 FB 粉專 ID

「通知完成」節點(選配)：
  YOUR_LINE_CHANNEL_ACCESS_TOKEN → LINE 的 channel token
  YOUR_LINE_USER_ID              → 要收通知的 LINE user id
  不想要通知就直接刪掉這個節點。

------------------------------------------------------------
【三、前置條件】
------------------------------------------------------------
  - FB 粉專 + IG 商業帳號(且 IG 已綁定到該粉專)
  - Meta 的 access token 要有發文權限
  - Threads 帳號 + 長期 access token
    (長期 token 約 60 天到期，建議搭配免費範本
     「Threads token 自動續期」自動延長)

------------------------------------------------------------
【四、小提醒：對外發文建議留一道人工確認】
------------------------------------------------------------
這條預設抓「最新一篇」就直接三平台發出。發出去就收不回，
建議實務上：稿子真的定稿了才把它變成「最新一篇」，或在前面
加一個你按了才往下走的確認步驟，避免半成品被自動發出去。

------------------------------------------------------------
【五、測試】
------------------------------------------------------------
先拿一篇測試稿手動執行一次，確認三平台都有出現、Sheet 的
「是否發布」有回填「是」，再決定要不要排程或接上自動觸發。

------------------------------------------------------------
【安全】
------------------------------------------------------------
所有 token 只存在你自己的 n8n，本範本不含任何人的真實金鑰，
也不會把資料送到任何第三方。
