【這個範本在做什麼】
每週一早上 8:00 自動抓三個來源，整理成 Google Sheet 一張
分區整合報告，並把每週數據往「趨勢」分頁右邊加一欄看成長：
  - Google Search Console：搜尋點擊 / 曝光 / 排名
  - GA4：造訪次數 / 訪客人數 / 瀏覽頁數
  - Threads(脆)：粉絲 / 瀏覽 / 互動

------------------------------------------------------------
【一、憑證：一把 Google 服務帳號(Service Account)就夠】
------------------------------------------------------------
所有節點都用同一把 googleApi 服務帳號憑證，需要開三個權限
(scope)：
  - webmasters.readonly   (GSC)
  - analytics.readonly    (GA4)
  - spreadsheets          (寫報告)

接好之後，把服務帳號的 email 分別授權出去，否則會 403：
  1. GSC：到 Search Console → 設定 → 使用者與權限，把服務
     帳號 email 加為「完整」使用者。
  2. GA4：到 GA4 管理 → 資源存取管理，把服務帳號 email 加為
     「檢視者」。(最常被忘記，忘了就一直 403)
  3. 試算表：直接把報告試算表「共用」給服務帳號 email(編輯者)。

------------------------------------------------------------
【二、要替換的佔位符(全部換成你自己的)】
------------------------------------------------------------
  YOUR_DOMAIN.com
    → 「GSC 查詢」節點網址裡，sc-domain 後面換成你的網域

  YOUR_GA4_PROPERTY_ID
    → 「GA4 查詢」節點網址裡，換成你的 GA4「資源 ID」
      (是資源 ID，不是帳戶 ID；在 GA4 管理 → 資源設定看)

  YOUR_GOOGLE_SHEET_ID
    → 出現在「寫入報告值 / 套色排版 / 讀趨勢 / 寫趨勢」四個
      節點網址，全部換成你的試算表 ID
      (網址 /spreadsheets/d/【這串】/edit)

  SHEET_ID(在「整合彙總」程式碼節點最上面 const SHEET_ID = 0)
    → 換成你「行銷整合報告」分頁的 gid 數字
      (點到那個分頁，網址 #gid=【這串數字】)

  YOUR_THREADS_USER_ID / YOUR_THREADS_ACCESS_TOKEN
    → 「Threads 粉絲」節點裡換成你的 Threads user id 與長期 token
      (長期 token 會過期，建議搭配另一個免費範本
       「Threads token 自動續期」一起用)

------------------------------------------------------------
【三、Google Sheet 準備兩個分頁】
------------------------------------------------------------
  1.「行銷整合報告」：放每週那張套色報告(工作流自動寫滿，
     你只要先建好空白分頁、記得它的 gid 填進上面 SHEET_ID)
  2.「趨勢」：放逐週累積的數據(工作流每週往右加一欄)

------------------------------------------------------------
【四、測試與排程】
------------------------------------------------------------
手動執行一次，看「行銷整合報告」分頁有沒有出現套色報告、
「趨勢」分頁有沒有新增一欄。沒問題就開成 Active，之後每週
一早上 8 點自動跑。時區已設 Asia/Taipei。

------------------------------------------------------------
【安全】
------------------------------------------------------------
金鑰、網域、token 只存在你自己的 n8n 與試算表，本範本不含
任何人的真實憑證，也不會把資料送到任何第三方。
